# 热处理考试复习资料 — 项目记录

## 项目概述

为用户构建热处理考试复习系统：一个可阅读的 Markdown 复习文档 + 一个交互式刷题网页。已推送到 https://github.com/MicroZhai/quiz ，通过 GitHub Pages 可在手机上使用。

## 文件结构

```
热处理复习资料/
├── 复习题整合.md    ← 📖 唯一复习文档，含全部679题及答案
├── quiz.html        ← 🧪 交互式刷题网页（单文件，零依赖）
├── index.html       ← 同 quiz.html，GitHub Pages 默认入口
├── questions.js     ← 题库数据，quiz.html 通过 <script> 加载
├── questions.json   ← 题库数据备份（JSON 格式）
├── parse_all.py     ← 从 复习题整合.md 重新解析生成题库
├── README.md        ← 项目说明
└── CLAUDE.md        ← 本文档：项目记录与维护指南
```

## 关键决策记录

### 1. 文档整合（2026-07-01）
- **决策**：5 个原始 .md → 1 个 `复习题整合.md`
- **原因**：用户要求只需一份复习资料，方便直接阅读
- **原始文件已删除**：`一、热处理基础知识.md` `二、热处理设备知识.md` `四、质量管理知识.md` `五、仪表知识.md` `复习资料.md`

### 2. 选择题答案恢复
- **决策**：`（ ）` → `（ D ）`，括号内填入正确答案字母
- **原因**：作为复习资料直接阅读时需要看到答案
- **匹配方式**：从 questions.json 读取答案，通过题干前 40 字标准化文本匹配
- **覆盖率**：282/270 题（部分题匹配到多个题干变体，部分题源数据无答案）

### 3. 判断题拆分
- **问题**：原始文件 ~21 行含多个判断题（一行 2-4 题）
- **处理**：以 `(标记)数字、` 模式为断点拆分为独立行
- **仍存在少量合并**：原始数据中 Q08/Q09 无标记分隔，无法完全自动拆分

### 4. 删除作图题
- **决策**：彻底删除作图题类型
- **原因**：用户明确表示"没有这个"
- **影响范围**：questions.json / questions.js / quiz.html / 复习题整合.md / parse_all.py

### 5. 目录导航
- **决策**：文档开头生成 Markdown 锚点目录
- **原因**：2400 行文档难以手动定位

### 6. GitHub Pages 部署
- **Repo**：https://github.com/MicroZhai/quiz
- **分支**：`main`（同时有 `master`，内容相同）
- **Pages 设置**：需手动开启 Settings → Pages → Deploy from branch → main + / (root)
- **访问地址**：https://microzhai.github.io/quiz/

## 维护指南

### 修改题目内容
1. 编辑 `复习题整合.md`
2. 运行 `python parse_all.py` 重新生成 questions.json 和 questions.js
3. 在浏览器打开 quiz.html 验证

### 选择题答案格式
- 正确答案写在括号内：`（ D ）`
- parse_all.py 会从 `（ D ）` 中提取答案字母
- 无答案的题保留空括号：`（ ）`

### 判断题格式
- 每题一行，格式：`(√) 01、题目内容` 或 `(×) 01、题目内容`
- parse_all.py 从行首 `(√)/(×)` 提取答案

### 填空题格式
- 题干中含 `______` 标记
- 答案在独立行：`   答：答案内容`
- parse_all.py 读取下一行的 `答：` 提取答案

### 添加新题
- 在对应章节的 `###` 题型区添加，按编号顺序
- 选择题：`N、题干（ X ）。` + 换行 + `A、选项1 B、选项2 ...`
- 判断题：`(√/×) N、题目内容`
- 填空题：`N、题目 ______` + 换行 + `   答：答案`

### 推送到 GitHub
```bash
git add -A
git commit -m "描述修改内容"
git push origin master:main
```

## 知识点标注系统（2026-07-01 新增）

### 概况
- 679 题全部标注了 `knowledge_point` 和 `knowledge_point_key` 字段
- 共 25 个核心知识点，覆盖率 ~95%
- 标注脚本：`tag_knowledge_points.py`（关键词匹配 + 规则）

### 知识点数据
- `knowledge_points.json`：25 个知识点的核心解析、关键概念、考试重点
- quiz.html 内嵌该数据，通过 `getKPInfo(question)` 获取题目对应知识点

### 更新题库流程
1. 修改 `复习题整合.md`
2. 运行 `python parse_all.py` 重新解析（可选，覆盖率不完整）
3. 运行 `python tag_knowledge_points.py` 为题库标注知识点
4. 运行 `python build_quiz.py` 重新生成 quiz.html（嵌入最新数据）
5. `cp quiz.html index.html` 同步

## quiz.html 技术要点

- 纯静态 HTML/CSS/JS，零依赖，file:// 协议可直接打开
- 题库通过 `<script src="questions.js">` 加载（`window.ALL_QUESTIONS`）
- 知识点数据内嵌在 HTML 中（`KNOWLEDGE_POINTS` / `KP_MAP`）
- **三个 Tab**：🧪 刷题 / 📖 复习资料 / 📋 学习建议
- **复习模式**：按知识点分组展示全部题目，答案可见，左侧知识点导航
- **学习建议页**：推荐学习路径、知识点题量分布、薄弱环节分析、高分技巧
- **刷题反馈**：答完题显示对应知识点解析（可点击展开）
- **考试模式**：按比例从题库抽题（largest remainder method），一页卷面，自动判分
- **填空题判分**：模糊匹配，关键词覆盖 ≥60% 算对
- **错题本**：localStorage 持久化存储
- **配色**：≥90 绿色 / ≥75 蓝色 / ≥60 橙色 / <60 红色

## 已知局限

1. 约 15 道选择题源数据无答案，保留空括号
2. 少量填空题 OCR 乱码未完全修复（晶粒、淬火等高频词已修复）
3. 综合复习章节的题目与前面章节有重复（属于汇总性质，刻意保留）
4. 判断题少量行仍存在格式问题（原始数据破损）
5. parse_all.py 从整合文档解析的覆盖率（538题）低于原始多文件解析（679题），因此以 questions.json 为准
6. ~30 道题（4.4%）未匹配到知识点标签，归入"综合/未分类"
